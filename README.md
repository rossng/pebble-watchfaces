# pebble-watchfaces

A monorepo of [Pebble](https://repebble.com) watchfaces. Most are built with the
**Alloy** SDK — Pebble's modern JavaScript framework (powered by Moddable's XS
engine), so those faces are written in **TypeScript** and run natively on the
watch. One face (`passing-trains`) drops to plain **native C** for framebuffer
access Alloy can't give.

Tooling is 2026-modern: **pnpm** workspaces, **TypeScript 6** (ESM watch code),
the **oxc** stack (**oxlint** + **oxfmt**) for lint/format, and a **Nix flake**
that pins the whole toolchain.

> **Status:** `pebble build` of the example is verified end-to-end — it produces
> a `simple-time.pbw` with both `emery` and `gabbro` binaries. Booting the
> emulator and installing to a watch need a GUI/USB and weren't run here, so those
> commands are from the official SDK guide. The canonical files for a new project
> always come from `pebble new-project --alloy`.

## Repository layout

```
pebble-watchfaces/
├── flake.nix                 # dev shell: node, pnpm, python, uv + emulator libs
├── nix/pebble-tool.nix       # experimental Nix package for the Pebble CLI
├── pnpm-workspace.yaml       # workspace = watchfaces/* + tools/*
├── tsconfig.base.json        # shared strict, ESM TypeScript config
├── .oxlintrc.json            # oxlint config
├── .oxfmtrc.json             # oxfmt config
├── types/pebble-alloy.d.ts   # ambient types for the embeddedjs runtime (Poco, watch, screen)
├── tools/
│   └── emu-tui/              # keyboard control panel for the running emulator
└── watchfaces/
    └── simple-time/          # ← the example watchface
        ├── package.json      #   Pebble manifest (`pebble` block) + scripts + typescript dep
        ├── tsconfig.json
        ├── wscript           #   waf build script (generated by `pebble new-project`)
        └── src/
            ├── c/mdbl.c      #   native entry point — boots the Moddable XS machine
            ├── pkjs/index.js #   phone-side (PebbleKit JS) code — plain JS, not TS
            └── embeddedjs/
                ├── main.ts   #   watch code (TypeScript, runs in XS on the watch)
                └── manifest.json  # Moddable manifest: modules + TS typings + skipLibCheck
```

Each watchface is a self-contained project: `pebble build` runs from the project
directory, and `package.json`'s `pebble` block is its app manifest. In an Alloy
face only `src/embeddedjs` is TypeScript; `src/pkjs` is bundled as plain
JavaScript by the build, so it stays `.js`. The native-C face (`passing-trains`)
instead puts its watch code in `src/c/**/*.c` — see its README for the shape.

## 1. Set up the toolchain

This repo uses a **git submodule** (train artwork) and **Git LFS** (binary resources
like screenshots and baked images). Right after cloning:

```bash
git submodule update --init        # fetch the ns-treinen artwork
git lfs install && git lfs pull    # fetch real PNG bytes (else they're LFS pointers)
```

You need the Pebble SDK (`pebble-tool` + a downloaded SDK) plus Node/pnpm. Pick
**one** of the following.

### Option A — Nix (recommended)

The flake provides a dev shell with Node 22, pnpm, Python, `uv`, and the native
libraries the emulator needs. It bootstraps `pebble-tool` with `uv` on first entry.

```bash
nix develop                  # or: direnv allow   (an .envrc is included)
pebble sdk install latest    # one-time SDK download (hundreds of MB)
pnpm install                 # workspace dependencies
```

> The Pebble _tool_ is installed via `uv` inside the shell (into `./.pebble-tooling`)
> because `pebble sdk install` downloads emulator/toolchain binaries at runtime,
> which doesn't fit a pure Nix derivation. `nix/pebble-tool.nix` is an experimental
> attempt to package the tool itself — see the file header before relying on it.

### Option B — Manual install

Per the [official SDK guide](https://developer.repebble.com/sdk/):

```bash
# Native deps — e.g. macOS:
brew install node uv
#            Ubuntu/WSL:
# sudo apt install nodejs npm libsdl2-2.0-0 libglib2.0-0 libpixman-1-0 zlib1g libsndio7.0

uv tool install pebble-tool   # needs Python 3.10+
pebble sdk install latest

corepack enable               # provides pnpm
pnpm install
```

Windows isn't supported directly — use WSL with the Ubuntu instructions.

## 2. Run a watchface in the emulator

From a watchface directory:

```bash
cd watchfaces/simple-time
pnpm dev          # build → install on the emery emulator → launch the control TUI
```

`pnpm dev` ends in [`emu-tui`](../../tools/emu-tui), a keyboard control panel for
the freshly-booted emulator: press `↑`/`↓`/`⏎`/`⌫` for the buttons, `t` to set the
clock, `f` to toggle 12h/24h, `a` for an accel tap, `p` for a screenshot, `q` to
quit. It just shells out to `pebble emu-*`, so the emulator window stays live.

Or step by step:

```bash
pebble build                      # compile → build/<name>.pbw (tsc runs via Moddable)
pebble install --emulator emery   # boot the emulator and install the face
node ../../tools/emu-tui/bin/emu-tui.js --emulator emery   # optional: drive it by keyboard
```

> Run builds through `pnpm` (`pnpm build` / `pnpm dev`), not a bare `pebble build`:
> Moddable shells out to `tsc`, and running via a pnpm script puts the watchface's
> local TypeScript on `PATH`. A bare `pebble build` only works if `tsc` is already
> on `PATH`.

> **Why `emu-tui` is launched by path, not as a workspace dependency.** `pebble
build` shells out to `npm install` against the watchface's own `package.json`,
> and npm can't parse pnpm's `workspace:` protocol. So a watchface manifest must
> stay npm-installable — never add `"…": "workspace:*"` deps to it. Shared tools
> under `tools/` are invoked directly (`node ../../tools/<tool>/bin/…`), which
> also keeps the working directory in the watchface (e.g. so `p` screenshots land
> there).

`emery` is the Pebble Time 2 emulator; Alloy also targets `gabbro` (Pebble Round 2).
Match the `--emulator` to a platform in the watchface's `targetPlatforms`.

Useful while iterating: `pebble logs` (stream device logs), `pebble screenshot`.

## 3. Deploy to a physical watch

The **Pebble Time 2** is the `emery` platform — already in `simple-time`'s
`targetPlatforms`, so `pebble build` produces an emery binary you can sideload.
(Pebble Round 2 is `gabbro`.) Match your watch to a platform in the manifest.

From the watchface directory:

1. `pnpm build` — produces `build/<name>.pbw` containing the binary for your watch.
2. Install the **Pebble** mobile app (<https://repebble.com/app>) and pair your watch.
3. In the app, enable **Developer Connection** (Settings → Developer); that screen
   shows the phone's IP.
4. Install over the local network or via the cloud:

   ```bash
   # Local network — phone relays to the watch over Bluetooth, so the computer
   # and phone must share a network. Use the IP from the Developer screen:
   PEBBLE_PHONE=192.168.1.42 pnpm deploy:phone
   # equivalently: pebble install --phone 192.168.1.42

   # …or route through CloudPebble, no shared network needed (sign in first):
   pebble login
   pnpm deploy:cloud          # pebble install --cloudpebble
   ```

The watchface is sideloaded onto the watch and appears in your face list. Stream
device logs while iterating with `pebble logs --phone <ip>`.

## 4. Add a new watchface

Generate a fresh Alloy project inside `watchfaces/` so its manifest matches the
current SDK, then it joins the pnpm workspace automatically:

```bash
cd watchfaces
pebble new-project --alloy my-face     # or: pnpm -w new my-face
```

This generates the `wscript`, `src/c/mdbl.c`, and `src/pkjs/index.js` that the
build requires. Then make it a TypeScript project the way `simple-time` is (use it
as the reference):

- Rename `src/embeddedjs/main.js` → `main.ts`.
- Add `"typescript": "^6.0.0"` to `devDependencies` (the build calls `tsc`).
- In `src/embeddedjs/manifest.json`, keep the `manifest_typings.json` include and
  add `"compilerOptions": { "skipLibCheck": true }` under
  `platforms.pebble.typescript.tsconfig` (see [the note below](#how-typescript-reaches-the-watch)).
- Add a `typecheck` script and a `tsconfig.json` that extends `../../tsconfig.base.json`
  and includes `../../types/**/*.d.ts`.
- **Do not** add `"type": "module"` to the watchface `package.json` — it breaks the
  pkjs webpack step, whose generated config is CommonJS.

Set `"watchapp": { "watchface": true }` in the `pebble` block. The package then
joins the pnpm workspace automatically.

## Workspace scripts

Run from the repo root:

```bash
pnpm typecheck     # tsc across every watchface
pnpm lint          # oxlint
pnpm format        # oxfmt (write); `pnpm format:check` to verify only
```

## How TypeScript reaches the watch

`pebble build` drives Moddable's build, which invokes `tsc` for you and generates
its own `tsconfig` from the `manifest.json` files. The `manifest_typings.json`
include in `src/embeddedjs/manifest.json` pulls in Moddable's Pebble type
definitions. The workspace's own `tsc`/`tsconfig.base.json` exist purely for
editor and CI type-checking (`noEmit`), backed by the ambient declarations in
[`types/pebble-alloy.d.ts`](types/pebble-alloy.d.ts) — those mirror just the watch
runtime surface the example uses, so the editor doesn't need the full SDK to be happy.

Two things that bit during bring-up and are worth knowing:

- **TypeScript 6, not 5.** SDK `4.9.169` generates a tsconfig with `target`/`lib`
  set to `es2025`, which only TypeScript ≥ 6 understands. (Older write-ups say
  "use 5.x, not 6.x" — that predates the `es2025` switch.) Each watchface pins
  `typescript@^6`.
- **`skipLibCheck` is required.** The SDK's own `pebble/global.d.ts` imports
  `web/websocket`, but its typings ship that module path-mapped as `websocket`, so
  `tsc` can't resolve it and fails _inside Moddable's own declarations_. The
  `manifest.json` injects `skipLibCheck: true` via
  `platforms.pebble.typescript.tsconfig.compilerOptions` to skip checking `.d.ts`
  files. Without it, no `.ts` watchface compiles.
